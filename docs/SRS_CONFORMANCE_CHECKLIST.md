# SRS Conformance Checklist

This checklist tracks the acceptance surface for the 27 SRS use cases. It is intentionally lightweight: each row names the permissions, audit events, persistence surface, happy path, and primary rejection paths that tests and implementation should cover before a UC is considered complete.

| UC | Required permissions | Required audit events | DB tables touched | Happy path to prove | Main rejection paths |
| --- | --- | --- | --- | --- | --- |
| UC1 Create custom role | `role.create` | Role created with actor and role snapshot | `roles`, `audit_logs` | Unique role created as `INACTIVE` | Missing permission, duplicate name, missing description |
| UC2 Assign permission to role | `role.permission.assign` | Before/after permission set | `roles`, `permissions`, `role_permissions`, `audit_logs` | Permission set replaced or added and role activates when first permission exists | Missing permission, unknown role, unknown permission, duplicate mapping |
| UC3 Assign/revoke staff role | `staff.role.assign` | Assignment/revocation with actor, target, reason | `users`, `roles`, `user_roles`, `audit_logs` | Active staff receives/revokes active role | Missing permission, inactive user, inactive role, duplicate assignment, missing revocation reason |
| UC4 View audit logs | `audit.read` | Audit read/export event where required | `audit_logs` | Filtered audit list and CSV export | Missing permission, invalid date range/filter |
| UC5 Add product and SKUs | `catalog.write` | Product/SKU/image before/after | `products`, `product_variants`, `product_images`, `product_categories`, `audit_logs` | Product with category, base price, image, and SKUs created | Missing permission, missing mandatory field, duplicate SKU, invalid price/image |
| UC6 Adjust SKU inventory | `inventory.adjust` plus override where needed | Stock before/after and reason | `inventory_balances`, `audit_logs` | Atomic adjustment records new balance | Missing permission, missing reason, negative stock without override |
| UC7 Transfer stock | `inventory.transfer` | Transfer status trail | `inventory_transfers`, `inventory_transfer_items`, `inventory_balances`, `audit_logs` | Draft, approve, receive, cancel update stock correctly | Missing permission, invalid state transition, insufficient stock |
| UC8 POS sale | POS sale permission or configured staff sale permission | Receipt/payment/stock decrement | `pos_shifts`, `pos_receipts`, `pos_receipt_items`, `payments`, `inventory_balances`, `audit_logs` | Open-shift sale creates receipt and decrements stock | Missing permission, no active shift, insufficient stock, payment failure |
| UC9 Process refund | `refund.process` | Refund/return status and stock disposition | `returns`, `return_items`, `refunds`, `payments`, `inventory_balances`, `audit_logs` | Eligible refund persists and updates status/stock | Missing permission, ineligible order/receipt, amount mismatch, duplicate refund |
| UC10 Manage discount codes | `promo.manage` | Discount before/after and redemption | `discount_codes`, `discount_redemptions`, `audit_logs` | Create/update/deactivate/validate discount | Missing permission, duplicate code, invalid dates, cap exceeded, min spend not met |
| UC11 Compose Instagram draft | `social.compose` | Draft create/update/submit/approve/publish | `instagram_drafts`, `product_images`, `audit_logs` | Product-derived draft with media rendition reaches publishable state | Missing permission, missing media, invalid status transition |
| UC12 Register account | Public | Registration and verification attempt | `users`, `audit_logs` | Unique customer registration creates verifiable account | Duplicate username/email/phone, weak password, invalid verification |
| UC13 Browse/filter/compare variants | Public | None unless analytics is required | `products`, `product_variants`, `categories`, `product_images` | Search/filter/compare returns matching variants | Invalid filter values, unavailable product hidden |
| UC14 AI recommendation chat | Authenticated customer or public per SRS | Recommendation request where required | `products`, `product_variants`, `inventory_balances`, optional chat log | Stock-aware recommendation can be added to cart | No stock, invalid prompt, recommended variant unavailable |
| UC15 Checkout and pay | Authenticated customer | Order/payment/reservation lifecycle | `carts`, `cart_items`, `orders`, `order_items`, `payments`, `inventory_balances`, `stock_reservations`, `discount_redemptions`, `audit_logs` | Reserve stock, apply discount, create order, confirm payment | No stock, payment failure, expired discount, reservation timeout |
| UC16 View stock by location | Authenticated customer/staff as applicable | None or stock view audit for staff | `inventory_balances`, `locations`, `product_variants` | Selected variant shows per-location availability | Unknown variant/location, inactive location |
| UC17 Track shipment | Authenticated customer/staff owner/read permission | Shipment status sync where required | `shipments`, `shipment_events`, `orders` | Shipment timeline reflects carrier events | Unknown order, unauthorized user, partner sync failure |
| UC18 Leave review with image | Authenticated customer | Review submitted/moderated | `reviews`, `orders`, `order_items`, `product_images` or review media | Delivered-order owner submits review with image | Not owner, undelivered order, outside review window, invalid image |
| UC19 View pending online orders | `order.read` | Staff order list read where required | `orders`, `order_items`, `payments`, `shipments` | Staff filters actionable orders | Missing permission, invalid filters |
| UC20 Mark order shipped | `order.ship` | Shipment/order before/after | `orders`, `shipments`, `shipment_events`, `audit_logs` | Shippable order gets shipment and tracking | Missing permission, invalid order state, invalid tracking |
| UC21 Connect Instagram | `social.connect` | Connect/reconnect/disconnect/token refresh | `instagram_connections`, `audit_logs` | OAuth state validates, scopes verified, token stored securely | Missing permission, invalid state, missing scope, token exchange failure |
| UC22 Purchase order/receiving | Procurement/inventory permission | PO approval and receipt | `purchase_orders`, `purchase_order_items`, `goods_receipts`, `goods_receipt_items`, `inventory_balances`, `audit_logs` | Approved PO receives goods and updates stock | Missing permission, unknown supplier, over-receipt, damaged stock handling |
| UC23 Allocate order to location | `order.read` or fulfillment allocation permission | Allocation and pick task state | `orders`, `order_items`, `order_allocations`, `stock_reservations`, `pick_tasks`, `audit_logs` | Order allocated to one or more locations with reservations | Missing permission, no stock, invalid split policy |
| UC24 Return intake/exchange | `refund.process` | Return/exchange lifecycle | `returns`, `return_items`, `refunds`, `orders`, `inventory_balances`, `audit_logs` | Eligible return inspected and dispositioned; exchange linked | Missing permission, ineligible item, duplicate return, invalid disposition |
| UC25 Cycle count/reconciliation | `inventory.adjust` or cycle count permission | Count create/submit/approve/reconcile | `cycle_counts`, `cycle_count_items`, `inventory_balances`, `audit_logs` | Snapshot count submitted and approved with reconciliation | Missing permission, mutable scope, threshold approval missing |
| UC26 POS shift close/cash reconciliation | POS shift permission or configured staff permission | Shift open/close/approval | `pos_shifts`, `cash_reconciliations`, `audit_logs` | Cashier closes shift and supervisor approves threshold variance | Missing permission, missing reason, self-approval over threshold |
| UC27 Tax invoice/adjustment note | Finance/invoice permission | Invoice/adjustment immutable chain | `tax_invoices`, `invoice_adjustments`, `audit_logs` | Invoice persists immutably and adjustment links to original | Missing permission, invalid tax fields, mutation after finalization |

## Completion Rule

A use case is complete only when implementation, backend tests, frontend behavior where applicable, and documentation agree with the row above and the detailed SRS business rules.
